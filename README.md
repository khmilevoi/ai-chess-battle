# AI Chess Battle

`AI Chess Battle` - локальное веб-приложение для шахматных матчей, где белые и
черные настраиваются независимо. Каждая сторона может быть человеком или
AI-актором на базе `OpenAI`, `Anthropic` или `Google`.

Приложение использует единый игровой цикл для всех типов акторов, показывает
легальные ходы на доске, сохраняет партии локально и позволяет возвращаться к
незавершенным матчам.

## Что умеет проект

- запускать `human vs AI`, `AI vs AI` и `human vs human`;
- настраивать акторов отдельно для белых и черных;
- показывать интерактивную доску с подсветкой доступных ходов;
- сохранять историю ходов и состояние матчей в браузере;
- открывать архив локально сохраненных партий;
- просматривать историю ходов и возвращаться к последней позиции;
- повторять ход после ошибки AI-актора;
- включать ручное подтверждение перед отправкой AI-запроса.

## Как запустить

### Требования

- `Node.js` `^20.19.0` или `>=22.12.0`
- `pnpm`

### Установка зависимостей

```bash
pnpm install
```

### Запуск в режиме разработки

```bash
pnpm dev
```

После запуска откройте адрес из вывода `Vite`, обычно это
`http://localhost:5173`.

### Production build

```bash
pnpm build
pnpm preview
```

## Как начать матч

1. Откройте страницу `Setup`.
2. Выберите актора для белых и для черных.
3. Если выбран `Human Actor`, дополнительных настроек не требуется.
4. Если выбран `OpenAI Actor`, `Anthropic Actor` или `Google Actor`, введите
   API key и выберите модель.
5. Нажмите `Start Match`.

Для локального запуска `.env` не нужен: ключи вводятся прямо в интерфейсе.

## Локальное хранение данных

- конфигурации акторов сохраняются в `localStorage`;
- активная партия и архив игр тоже сохраняются в `localStorage`;
- API-ключи AI-провайдеров хранятся локально в браузере.

Это удобно для разработки, но не подходит для production-хранения секретов.
